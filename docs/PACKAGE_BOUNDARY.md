# حدود الحزمة

> هذه وثيقة مرجعية.
>
> للبداية التشغيلية:
>
> ```text
> docs/START_HERE.md
> ```

## التعريف

يوجد في المستودع حدّان مقصودان ومتكاملان:

1. الحزمة الجذرية
2. حزمة المحرك الداخلي

## الحزمة الجذرية

الموقع:

```text
package.json
```

المسؤولية:

- تشغيل الاختبارات
- توليد الوثائق
- فحص العقود
- التحقق من اتساق المستودع

## حزمة المحرك الداخلي

الموقع:

```text
.Systematize/scripts/node/package.json
```

المسؤولية:

- تعريف محرك
  `Node CLI`
- توفير نقطة تشغيل
  `syskit`
- تجميع أوامر التنفيذ الرسمية

## القاعدة

- الجذر ليس محرك تشغيل المنتج.
- الحزمة الداخلية ليست مساحة تحقق المستودع.
- العلاقة بينهما مقصودة:
  الجذر يختبر ويولد،
  والداخل ينفذ.

## أثر هذا الحد

- يمنع خلط واجبات
  التحقق
  مع واجبات
  التنفيذ
- يجعل نشر المحرك الداخلي ممكنًا من دون نقل كل أدوات التحقق
- يبقي معمارية المستودع منضبطة حتى مع توسع الوثائق والاختبارات
