# التوزيع الرسمي

هذه الوثيقة تضبط المسار الرسمي لبناء حزمة التوزيع الخاصة بالإطار بعد إعادة التموضع.

## الناتج

الأمر الرسمي:

```text
npm run package:dist
```

هذا المسار يمر عبر:

```text
node .Systematize/scripts/node/cli.mjs build-distribution
```

وقبل بناء الحزمة يقوم المحرك نفسه بتوليد الوثائق الرسمية المطلوبة من المصدر.

ينتج شيئين داخل:

```text
dist/
```

- مجلد توزيع كامل:
  `dist/systematize-framework/`
- حزمة تشغيل:
  ملف
  `.tgz`
  لمحرك
  `Node`

## عقد التثبيت

داخل مجلد التوزيع يوجد مساران رسميان:

```text
install-framework.mjs
install-framework.ps1
```

وكلاهما يشغّل التهيئة الرسمية عبر محرك:

```text
.Systematize/scripts/node/cli.mjs
```

## القاعدة

- التوزيع الرسمي يبني حزمة التشغيل، والوثائق الرسمية، والقوالب، والسكريبتات، وكتالوج حزم التوسعات.
- محتويات التوزيع المعتمدة تُقرأ من:
  `.Systematize/config/distribution-manifest.json`
  ثم تُثبت داخل الحزمة الناتجة عبر:
  `distribution-manifest.json`
- التثبيت الرسمي لا ينسخ منطقًا موازيًا؛ بل يمر دائمًا عبر محرك
  `Node`
  الرسمي.
- التوسعات الافتراضية تُثبت من الكتالوج أثناء
  `init`
  وفق بيانات الحزمة نفسها.
